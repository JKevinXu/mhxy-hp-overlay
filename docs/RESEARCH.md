# Research: 梦幻西游手游可见血量显示工具

## 原始需求拆解

用户想要“实时的梦幻西游血量显示工具”。这个需求有两种解释：

1. 不合规解释：读取游戏/模拟器内存、网络包或内部状态，显示真实但画面不可见的 HP。
2. 合规解释：只从用户授权的屏幕画面、截图、录屏中提取肉眼可见的血条/血量文本，并显示估计结果。

本仓库采用第 2 种解释。

## 合规边界

禁止：

- 内存读取/写入。
- 封包监听、协议解析、私有 API 逆向。
- 客户端注入、hook、修改资源或绕过检测。
- 自动点击、自动输入、自动战斗、自动任务。
- 显示隐藏血量、隐藏冷却、隐藏坐标等不可见状态。

允许：

- 用户授权截图/屏幕录制。
- 离线截图/视频分析。
- 可见血条颜色检测。
- 可见文本 OCR/正则解析。
- 手动 ROI 校准。
- 本地 debug 图和本地结果展示。

## GameAutomation 可借鉴点

读取了 `/Users/kx/ws/GameAutomation` 后，适合借鉴：

- OpenCV + Pillow/Numpy 图像处理技术栈。
- Retina 显示下 logical/physical 坐标区分。
- debug 图片保存与置信度日志。
- MuMu 模拟器作为屏幕画面来源。

明确不迁移：

- pyautogui 点击。
- LaunchAgent 自动化调度。
- 师门任务工作流。
- MCP 触发自动游戏行为。

## MVP 技术路线

1. 第一阶段：静态图片 ROI 检测。
   - 用 HSV 阈值提取红色血条区域。
   - 计算红色像素沿 x 方向覆盖比例。
   - 输出 `hp_ratio` 与 `confidence`。

2. 第二阶段：屏幕实时采样。
   - 使用 `mss` 截取用户指定 ROI。
   - 按固定 interval 分析。
   - 只在终端显示，不覆盖/修改游戏窗口。

3. 第三阶段：校准与 UI。
   - 保存多个 ROI：我方、队友、目标、怪物。
   - 终端 dashboard 或独立透明窗口。
   - 每个显示项标注 “visible estimate”。

## 风险与缓解

- 血条颜色可能变化：配置 HSV 阈值，保存 debug mask。
- 遮挡/动画导致误判：显示置信度和最近 N 帧平滑值。
- 数字 OCR 不稳定：优先血条比例，文本 OCR 作为可选增强。
- 实时工具可能被误用：README、设计文档、代码 guard 明确禁止隐藏状态和自动输入。
